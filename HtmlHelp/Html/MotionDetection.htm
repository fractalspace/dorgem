Motion Detection

From version 1.1.0 Dorgem has motion detection. The initial implementation is has a known limitation.

The motion is always compared to the previous capture of the camera, even if it is used by another store event. To change this requires too much work at this moment.

When 'Motion Detection' is turned on, an image will only be stored, when there is a difference between the last stored image and the current snapshot.

To use the camera for security set the interval low, so the images can be compared often and use an archive AVI.

Pictures requested by the web server are always servered directly from the camera. Motion detection is bypassed by the web server.
